# Developer-Experience (DX) & Inner-Source

- **Development Environment & Tooling**

  - IDE mastery: VS Code, IntelliJ, vim/neovim, extensions, customization, productivity
  - Dotfiles management: configuration versioning, cross-platform compatibility, automation
  - Language servers: LSP protocol, code intelligence, refactoring, error detection
  - Terminal optimization: shell configuration, command-line tools, productivity shortcuts
  - Local development: environment setup, dependency management, service orchestration
  - Debugging tools: debugger configuration, remote debugging, performance profiling
  - Code navigation: symbol search, cross-references, call hierarchies, dependency graphs

- **Repository Management & Standards**

  - Repository structure: monorepo vs polyrepo, directory organization, naming conventions
  - Commit hygiene: conventional commits, atomic commits, meaningful messages, history
  - Pre-commit hooks: code formatting, linting, security scanning, test execution
  - Branch protection: required reviews, status checks, merge strategies, permissions
  - Template repositories: project scaffolding, best practices, standardization
  - Documentation: README standards, API documentation, architectural decisions
  - License management: open source licenses, compliance, dependency auditing

- **Code Discovery & Knowledge Management**

  - Code search: Sourcegraph, GitHub search, IDE search, semantic search, indexing
  - Knowledge graphs: code relationships, dependency visualization, impact analysis
  - Documentation systems: wikis, knowledge bases, API documentation, runbooks
  - Code reviews: review guidelines, automated checks, knowledge sharing, mentoring
  - Onboarding: developer guides, setup automation, learning paths, mentorship
  - Tribal knowledge: knowledge capture, documentation, video tutorials, workshops
  - Community building: tech talks, guilds, communities of practice, knowledge sharing

- **Platform Engineering & Golden Paths**
  - Platform teams: developer productivity, infrastructure abstraction, self-service
  - Golden paths: opinionated workflows, best practices, standardization, compliance
  - Developer portals: service catalogs, documentation, metrics, self-service capabilities
  - Paved roads: pre-configured environments, deployment pipelines, monitoring setup
  - Scaffolding tools: project templates, code generators, boilerplate automation
  - Internal tooling: custom tools, automation, productivity enhancements, integration
  - Metrics: DORA metrics, developer satisfaction, productivity measurement, improvement
